# Backlog post-finalisation

## Back-office editorial

Action a traiter apres finalisation du site vitrine :

- Creer un back-office pour gerer les vehicules, les blocs de pages, les actualites, les evenements, les partenaires et les medias.
- Option recommandee : Directus ou Strapi en CMS headless, avec site statique ou front connecte a l'API.
- Collections a prevoir : `vehicles`, `vehicle_categories`, `pages`, `page_blocks`, `events`, `news`, `partners`, `menus`, `media`, `settings`, `translations`.
- Objectif : permettre au client de modifier les contenus, images, fiches vehicules, blocs de homepage et versions FR/EN sans toucher au code.
